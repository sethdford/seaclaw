SeaClaw supports additional messaging channels. Each requires its own build flag and configuration.

## IRC

Build: `-DSC_ENABLE_IRC=ON`

Connect to IRC servers. Configure server, port, nick, and channels.

## Matrix

Build: `-DSC_ENABLE_MATRIX=ON`

Connect to Matrix.org or self-hosted Synapse. Requires homeserver URL, user ID, access token, and room IDs.

## WhatsApp

Build: `-DSC_ENABLE_WHATSAPP=ON`

Uses WhatsApp Business API. Webhook path: `/webhook/whatsapp` or `/whatsapp`. Requires Business API credentials and webhook verification.

## LINE

Build: `-DSC_ENABLE_LINE=ON`

LINE Messaging API. Webhook path: `/webhook/line` or `/line`. Requires Channel Secret and Channel Access Token.

## DingTalk

Build: `-DSC_ENABLE_DINGTALK=ON`

DingTalk (钉钉) custom bot. Configure webhook URL and secret.

## Lark / Feishu

Build: `-DSC_ENABLE_LARK=ON`

Lark (Feishu 飞书) app. Webhook path: `/webhook/lark` or `/lark`. Requires App ID, App Secret, and event subscription.

## OneBot

Build: `-DSC_ENABLE_ONEBOT=ON`

[OneBot](https://github.com/botuniverse/onebot) protocol (e.g. go-cqhttp, Lagrange). HTTP API or reverse WebSocket.

## QQ

Build: `-DSC_ENABLE_QQ=ON`

QQ (腾讯QQ) bot integration. Configure via official QQ bot platform.

## Email

Build: `-DSC_ENABLE_EMAIL=ON`

Bidirectional email: send via SMTP and receive via IMAP polling.

### Config

```json
{
  "channels": {
    "email": {
      "smtp_host": "smtp.gmail.com",
      "smtp_port": 587,
      "from_address": "bot@gmail.com",
      "smtp_user": "bot@gmail.com",
      "smtp_pass": "your-app-password",
      "imap_host": "imap.gmail.com",
      "imap_port": 993
    }
  }
}
```

Inbound: polls IMAP for unseen messages every 30 seconds. Outbound: sends via SMTP with TLS.

## iMessage

Build: `-DSC_ENABLE_IMESSAGE=ON`

Bidirectional Apple iMessage on macOS. Sends via AppleScript, receives by polling `~/Library/Messages/chat.db` (requires Full Disk Access).

### Config

```json
{
  "channels": {
    "imessage": {
      "default_target": "+15551234567"
    }
  }
}
```

Inbound: polls the Messages database every 3 seconds. Outbound: sends via osascript.

## Mattermost

Build: `-DSC_ENABLE_MATTERMOST=ON`

Mattermost server integration. Similar to Slack: bot token, server URL, and channel configuration.

## MaixCam

Build: `-DSC_ENABLE_MAIXCAM=ON`

MaixCam board channel for embedded/hardware deployments.

## Web / WebSocket

Build: `-DSC_ENABLE_WEB=ON`

WebSocket-based web UI channel. Clients connect to the gateway over WSS. Supports streaming and real-time chat.

## Configuration pattern

Each channel expects a `channels.<name>` entry in config, typically an array of account configs:

```json
{
  "channels": {
    "irc": [{ "server": "irc.libera.chat", "nick": "seaclaw" }],
    "matrix": [
      { "homeserver": "https://matrix.org", "user_id": "@bot:matrix.org" }
    ],
    "web": []
  }
}
```

Refer to the source in `src/channels/` for each channel's exact config schema.
