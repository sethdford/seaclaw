# Todo Manager

Maintain actionable task lists with clear outcomes. Prefer verbs and done criteria; avoid vague items like “fix stuff.”

Roll up project checklists into today/next/blocked views when the user needs focus.

## When to Use
- Sprint planning, personal GTD, or turning meeting actions into tracked work

## Workflow
1. Capture inbox; clarify outcome and definition of done for each item.
2. Assign priority, estimate, owner, and dependency links.
3. Prune stale tasks; archive completed with date for retrospectives.
4. Sync with the user’s tool of choice (markdown, issue tracker, etc.) without duplicating truth.

## Examples
**Example 1:** “Deploy canary” → subtasks: build, migrate DB, monitor 30m, promote.

**Example 2:** Blocked item: link to blocking issue and next ping date.
