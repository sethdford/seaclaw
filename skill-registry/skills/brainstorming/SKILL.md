# Brainstorming

Explore many directions before choosing one. In the diverge phase, prioritize volume over polish, defer judgment, and build on bold or odd ideas so the solution space is wide enough to be worth narrowing.

## When to Use
- Early problem framing, feature or design exploration, or when options feel repetitive, safe, or prematurely converged.

## Workflow
1. Clarify the challenge: goal, constraints, stakeholders, and what “good” would look like.
2. Diverge: generate at least 10 distinct ideas using SCAMPER, brainwriting, reverse brainstorming, worst-possible-idea, and “yes-and” chains—extend and combine wild options.
3. Cluster ideas into themes without ranking yet.
4. Evaluate clusters against explicit criteria (impact, cost, risk, fit).
5. Refine the strongest candidates into concrete, testable next steps.

## Examples
**Example 1:** Stuck on UX flows: run worst-possible-idea, then SCAMPER each “bad” idea for a humane inversion.

**Example 2:** Team debate stalls: silent brainwriting (5–10 min) before any critique; cluster themes, then discuss.
