SeaClaw's motion system applies **Pixar's 12 principles of animation** to UI interactions. Every animation has purpose — it guides attention, confirms actions, and creates spatial coherence.

## Core Durations

| Token              | Value | CSS Variable            | Usage                             |
| ------------------ | :---: | ----------------------- | --------------------------------- |
| `duration.instant` | 100ms | `--sc-duration-instant` | Hovers, state toggles             |
| `duration.fast`    | 150ms | `--sc-duration-fast`    | Button presses, small reveals     |
| `duration.normal`  | 250ms | `--sc-duration-normal`  | Panel transitions, fades          |
| `duration.slow`    | 400ms | `--sc-duration-slow`    | Modal entrances, page transitions |
| `duration.slower`  | 600ms | `--sc-duration-slower`  | Complex orchestrated sequences    |

## Micro-Physics Presets (Pixar Principles)

These presets encode Pixar animation principles as token-driven CSS keyframes.

### Button Press — Squash & Stretch

When pressed, buttons compress slightly (squash), then spring back with overshoot (stretch).

| Token                                   | Value   | Principle              |
| --------------------------------------- | ------- | ---------------------- |
| `micro-physics.button-press.scale`      | `0.96`  | Squash on press        |
| `micro-physics.button-press.duration`   | `80ms`  | Fast physical response |
| `micro-physics.button-release.scale`    | `1.02`  | Stretch on release     |
| `micro-physics.button-release.duration` | `200ms` | Slower recovery        |

```css
.btn:active {
  transform: scale(var(--sc-physics-button-press-scale));
  transition: transform var(--sc-physics-button-press-duration)
    var(--sc-ease-spring);
}
```

### Modal Anticipation

Before appearing, modals briefly scale down (anticipation), then spring to full size.

| Token                                       | Value   | Principle            |
| ------------------------------------------- | ------- | -------------------- |
| `micro-physics.modal-anticipation.scale`    | `0.95`  | Preparatory pullback |
| `micro-physics.modal-anticipation.duration` | `120ms` | Brief setup          |

### Card Hover — Follow Through

Cards lift slightly with a springy overshoot on hover, settling to final position.

| Token                                  | Value   | Principle    |
| -------------------------------------- | ------- | ------------ |
| `micro-physics.card-hover.translate-y` | `-2px`  | Subtle lift  |
| `micro-physics.card-hover.duration`    | `300ms` | Gentle float |

### Dropdown Anticipation

Dropdowns compress before expanding, giving a sense of stored energy.

| Token                                          | Value   | Principle            |
| ---------------------------------------------- | ------- | -------------------- |
| `micro-physics.dropdown-anticipation.scale-y`  | `0.95`  | Vertical compression |
| `micro-physics.dropdown-anticipation.duration` | `100ms` | Quick wind-up        |

### Toggle Squash

Toggle switches squash horizontally during state change.

| Token                                  | Value   | Principle         |
| -------------------------------------- | ------- | ----------------- |
| `micro-physics.toggle-squash.scale-x`  | `0.9`   | Horizontal squash |
| `micro-physics.toggle-squash.duration` | `150ms` | Snappy            |

## Utility Classes

### `.sc-physics-press:active`

Applies button press squash/stretch on any interactive element:

```html
<button class="sc-physics-press">Click me</button>
```

### `.sc-physics-hover`

Applies card hover lift with follow-through:

```html
<div class="sc-physics-hover">Hoverable card</div>
```

### `.sc-anticipate-enter`

Applies anticipation entrance animation on mount:

```html
<dialog class="sc-anticipate-enter">Modal content</dialog>
```

## Reduced Motion

All animations respect `prefers-reduced-motion: reduce`. When active:

- Durations are set to `0ms`
- Transforms are removed
- Only opacity transitions remain (they don't cause motion sickness)

```css
@media (prefers-reduced-motion: reduce) {
  .sc-physics-press:active,
  .sc-physics-hover:hover {
    transform: none !important;
    transition: none !important;
  }
}
```

## When to Animate

| Scenario         | Duration  | Easing     | Principle        |
| ---------------- | --------- | ---------- | ---------------- |
| Button feedback  | `instant` | `spring`   | Squash & stretch |
| Card hover       | `normal`  | `spring`   | Follow-through   |
| Modal entrance   | `slow`    | `bounce`   | Anticipation     |
| Page transition  | `slow`    | `ease-out` | Staging          |
| Toast appearance | `fast`    | `ease-out` | Timing           |
| Loading spinner  | `slower`  | `linear`   | Secondary action |

## When NOT to Animate

- Scrolling content into view (use CSS `scroll-behavior: smooth` instead)
- Color theme transitions (instant swap, no fade)
- Error state changes (immediate, no delay)
- Content that hasn't changed (don't re-animate)
