# Pomodoro Timer

Structure focus sessions with explicit breaks to reduce context switching. Default 25/5 is a starting point; adapt to meeting calendars and deep-work needs.

Track interruptions honestly; reschedule rather than silently skipping breaks.

## When to Use
- Deep work blocks, study sessions, or breaking large tasks into timed slices

## Workflow
1. Choose task, define done criteria for the pomodoro, mute notifications.
2. Run timer; if interrupted, mark void or partial per user preference.
3. Take break away from screen; after 4 cycles, prefer a longer break.
4. Log completed pomodoros for velocity trends if useful.

## Examples
**Example 1:** Two pomodoros on code review, one on implementation.

**Example 2:** Shorten to 15m when between meetings.
