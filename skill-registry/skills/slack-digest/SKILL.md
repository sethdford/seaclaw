# Slack Digest

Summarize channel activity for people catching up. Respect workspace retention and confidentiality; don’t paste DMs or private channels without scope.

Highlight threads needing response and decisions made.

## When to Use
- End-of-day summaries, #general noise reduction, or project channel rollups

## Workflow
1. Scope channels and time window; filter bots and low-signal integrations.
2. Cluster by thread/topic; count reactions as weak priority signals.
3. Pull out @mentions and action requests explicitly.
4. Link to Slack threads for context; keep digest under readable length.

## Examples
**Example 1:** “5 threads discussed API redesign; decision: adopt option B—see thread.”

**Example 2:** List unanswered questions with suggested owners from thread participants.
