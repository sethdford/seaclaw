# Design Detections Command

Chain these steps:

1. Use `siem-rule-design` to design SIEM rules for target threats
2. Use `detection-engineering` to build detection logic and test against known attack patterns
3. Use `log-analysis-security` to analyze logs and refine detection logic
4. Use `threat-intelligence-integration` to incorporate threat intel into detections

Deliverables:

- SIEM rule definitions for priority threats
- Detection logic with testing methodology
- False positive baseline and tuning plan
- Threat intelligence integration plan
- Detection coverage summary (% of attack techniques detected)

After completion, suggest follow-up commands: `manage-vulnerabilities`, `setup-monitoring`.
