# Meeting Notes

Transform raw notes or transcripts into decisions, rationale, and owned action items. Distinguish agreed outcomes from parking-lot ideas.

Use consistent IDs for actions (owner + due date). Flag ambiguities instead of inventing consensus.

## When to Use
- After live meetings, async decision threads, or when cleaning up shared doc chaos

## Workflow
1. Capture attendees, agenda, and timebox if known.
2. Extract **Decisions** (what changed), **Discussion** (brief), **Actions** (verb + owner + due).
3. Note open questions and who must answer them.
4. Distribute in the channel the team uses; link recordings only with permission.

## Examples
**Example 1:** “DECISION: Ship v2 behind flag by Friday. ACTION: @alex load test by Wed.”

**Example 2:** No decision: “OPEN: pricing model—needs finance input by next sync.”
