# Meeting Facilitation

Guide the user to run meetings that earn their cost: clear intent, tight structure, respectful time use, and traceable follow-through.

## When to Use
- Planning or running syncs, decision forums, brainstorms, or recurring team rituals.

## Workflow
1. **State purpose** up front: decide, discuss, inform, or brainstorm—one primary mode.
2. **Use an agenda with time boxes**; name a facilitator and a note-taker (can be the same person only if the group is tiny).
3. **Open with the goal** in one sentence; keep discussion tied to it.
4. **Park tangents** explicitly: “Strong point—let’s capture it for [owner/topic] and return to the agenda.”
5. **Close with**: decisions made, **action items** (who / what / by when), and open questions.
6. **Challenge first**: could this be an email or async thread? **Cancel** if there is no unique value in realtime.

## Examples
**Example 1:** Status meeting drifting → restate purpose, table deep dives to a follow-up with only affected people.

**Example 2:** Brainstorm scheduled but leadership already decided → rename as “alignment on rollout” or reschedule to respect everyone’s time.
