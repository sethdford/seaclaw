---
title: Memory Migration
description: Migrate memory entries between SQLite and Markdown backends.
---

SeaClaw provides a migration tool to transfer memory entries between backends. Use it when switching storage (e.g. SQLite → Markdown) or consolidating data.

## CLI usage

```bash
seaclaw migrate [options] [target_path]
seaclaw migrate sqlite <source_db_path> [target_path]
seaclaw migrate markdown <source_directory> [target_path]
```

### Examples

**Migrate from SQLite to Markdown (dry run):**

```bash
seaclaw migrate sqlite ~/.seaclaw/memory.db ./memories
```

**Migrate from Markdown to SQLite:**

```bash
seaclaw migrate markdown ./old-memories ~/.seaclaw/new-memory.db
```

**Target path only** (migrate from empty source to target; useful for dry run):

```bash
seaclaw migrate ./output-dir
```

## Supported paths

| Source   | Target   | Description                   |
| -------- | -------- | ----------------------------- |
| SQLite   | Markdown | Export DB to Markdown files   |
| Markdown | SQLite   | Import Markdown files into DB |
| None     | Markdown | Empty run (0 entries)         |

## How it works

1. Opens the source backend (SQLite DB or Markdown directory)
2. Lists all entries via the memory vtable `list`
3. For each entry, calls `store` on the target backend
4. Tracks imported, skipped, and error counts

## Migration stats

After migration, the CLI prints:

```
Migration: N from sqlite, M from markdown, K imported
```

- `from_sqlite` / `from_markdown`: Entries read from source
- `imported`: Successfully written to target
- `skipped`: Entries with missing key or content
- `errors`: Store failures

## Dry run

The current implementation defaults to `dry_run: true` when source is `SC_MIGRATION_SOURCE_NONE`. For actual migrations with a real source, entries are written to the target. Check the implementation for exact dry-run behavior.

## SQLite requirement

When SQLite is disabled (`SC_ENABLE_SQLITE` off), only Markdown ↔ Markdown and None → Markdown migrations are supported. SQLite as source or target returns `SC_ERR_NOT_SUPPORTED`.
