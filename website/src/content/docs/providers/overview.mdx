SeaClaw uses a **vtable-driven provider interface**. All AI model backends implement `sc_provider_t` and are selected by name at runtime.

## Provider vtable interface

Each provider implements:

| Method                  | Description                                    |
| ----------------------- | ---------------------------------------------- |
| `chat`                  | Non-streaming chat completion                  |
| `chat_with_tools`       | Chat with tool calls (optional)                |
| `stream_chat`           | Streaming completion (optional)                |
| `supports_native_tools` | Whether provider uses OpenAI-style tool schema |
| `get_name`              | Provider identifier                            |
| `deinit`                | Cleanup                                        |

Optional methods: `warmup`, `supports_streaming`, `supports_vision`.

## Selecting a provider

Set `default_provider` in config or pass `--provider` on the CLI:

```bash
./seaclaw agent --provider ollama --model llama3
```

For compatible providers, add a `providers` entry with your API key:

```json
{
  "providers": [{ "name": "groq", "api_key": "gsk_..." }],
  "default_provider": "groq"
}
```
