---
title: Cloud Providers
description: Configure cloud AI providers (OpenAI, Anthropic, Gemini, etc.).
---

SeaClaw supports major cloud AI providers. Configure each with an API key in `~/.seaclaw/config.json` or via environment variables.

## OpenAI

```json
{
  "default_provider": "openai",
  "default_model": "gpt-4o",
  "providers": [
    {
      "name": "openai",
      "api_key": "sk-proj-..."
    }
  ]
}
```

Or use the environment variable:

```bash
export OPENAI_API_KEY="sk-proj-..."
./seaclaw agent
```

Models: `gpt-4o`, `gpt-4o-mini`, `gpt-4-turbo`, etc.

## Anthropic

```json
{
  "default_provider": "anthropic",
  "default_model": "claude-sonnet-4-20250514",
  "providers": [
    {
      "name": "anthropic",
      "api_key": "sk-ant-..."
    }
  ]

```

Environment variable:

```bash
export ANTHROPIC_API_KEY="sk-ant-..."
```

Models: `claude-sonnet-4-20250514`, `claude-opus-4`, `claude-3-5-haiku`, etc.

## OpenRouter

OpenRouter provides a single API for many models:

```json
{
  "default_provider": "openrouter",
  "default_model": "anthropic/claude-sonnet-4",
  "providers": [
    {
      "name": "openrouter",
      "api_key": "sk-or-v1-..."
    }
  ]
}
```

Environment variable: `OPENROUTER_API_KEY`

Models use the format `provider/model`, e.g. `anthropic/claude-sonnet-4`, `openai/gpt-4o`, `meta-llama/llama-3.1-70b-instruct`.

## Base URL override

Any provider can use a custom base URL:

```json
{
  "providers": [
    {
      "name": "openai",
      "api_key": "sk-...",
      "base_url": "https://your-proxy.example.com/v1"
    }
  ]
}
```

Useful for proxies, Azure OpenAI, or self-hosted endpoints.
