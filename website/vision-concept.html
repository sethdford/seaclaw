Core Philosophy

Every section has ONE job. One insight. One emotion.
Typography IS the visual. The numbers ARE the product shot.
Progressive disclosure — start simple, reveal complexity on scroll.

What Makes This "Apple-tier"

Narrative arc: Vision → Proof → Problem → Contrast → Depth → Ecosystem → Data → Action. Not a feature list — a story.

Typography as hero: "118×" in massive display type IS the product screenshot. h-uman's advantage is invisible (small binary) — so we make the NUMBER the visual.

Whitespace is deliberate: 120px+ section padding. Content breathes. Nothing competes for attention.

Two-paragraph hero: First paragraph explains what h-uman DOES (plain English). Second gives specs. Visitors understand value before they see stats.

Key Improvements Over Current

1. Hero explains WHAT h-uman does (not just how small it is)

2. Install command with copy button right in the hero

3. Version badge signals active development

4. Comparison callout (118×/50×/Zero) appears EARLY, right after the problem

5. Bento grid replaces flat 4-column card list — visual hierarchy through asymmetry

6. Ecosystem as 3 category cards with big numbers — not a flat pill cloud

7. CTA is differentiated: "Ship your first agent" (action) vs hero's "Bring AI to every device" (vision)

8. Card metrics — subtle data points in card corners reward careful readers

SOTA Patterns Used

• Apple: Massive type, progressive disclosure, numbers as visuals, aggressive whitespace

• Linear: Bento grid, glass-less dark cards, spring hover animations

• Stripe: Calm confidence, Tufte-style data viz, clean hierarchy

• Vercel: Install bar in hero, monospace details, speed metrics

• Raycast: Badge-style metadata, hover glow borders, metric callouts
