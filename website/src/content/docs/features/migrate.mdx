## Migration from OpenClaw

SeaClaw is a drop-in replacement for [OpenClaw](https://github.com/openclaw/openclaw) (TypeScript, Node.js). It preserves your memory, config structure, and channel integrations while dramatically reducing resource usage.

| Dimension        | OpenClaw               | SeaClaw                        |
| ---------------- | ---------------------- | ------------------------------ |
| **Binary size**  | ~28 MB (npm dist)      | **430 KB**                     |
| **Memory**       | Node.js heap (100+ MB) | **&lt; 5 MB** peak RSS         |
| **Dependencies** | 1,200+ npm packages    | **0** — libc + optional SQLite |
| **Secrets**      | Plain-text API keys    | ChaCha20-Poly1305 encrypted    |

### 1. Install SeaClaw

<Tabs>
  <TabItem label="Build from source">
    ```bash git clone https://github.com/sethdford/seaclaw.git cd seaclaw &&
    mkdir -p build && cd build cmake .. -DCMAKE_BUILD_TYPE=MinSizeRel

    -m) chmod +x seaclaw && sudo mv seaclaw /usr/local/bin/ ```
  </TabItem>
</Tabs>

### 2. Migrate memory

SeaClaw imports memories directly from OpenClaw's SQLite brain database:

```bash
seaclaw migrate sqlite ~/.openclaw/brain.db
```

The migration tool auto-detects common schema variants and maps columns automatically.

### 3. Migrate config

SeaClaw uses the **same config structure** as OpenClaw. Most keys map 1:1.

**OpenClaw** (`~/.openclaw/config.json`):

```json
{
  "default_provider": "openai",
  "default_model": "gpt-4o",
  "providers": [{ "name": "openai", "api_key": "sk-proj-xxx" }]
}
```

**SeaClaw** (`~/.seaclaw/config.json`):

```json
{
  "models": {
    "providers": { "openai": { "api_key": "sk-proj-xxx" } }
  },
  "agents": {
    "defaults": { "model": { "primary": "gpt-4o" } }
  }
}
```

### 4. Verify

```bash
seaclaw doctor          # Config, provider, memory OK?
seaclaw status          # Full system status
seaclaw agent -m "What do you remember about me?"
```

## Memory Backend Migration

Transfer memory entries between SQLite and Markdown backends when switching storage.

### CLI usage

```bash
seaclaw migrate sqlite <source_db_path> [target_path]
seaclaw migrate markdown <source_directory> [target_path]
```

### Examples

**SQLite → Markdown:**

```bash
seaclaw migrate sqlite ~/.seaclaw/memory.db ./memories
```

**Markdown → SQLite:**

```bash
seaclaw migrate markdown ./old-memories ~/.seaclaw/new-memory.db
```

### Supported paths

| Source   | Target   | Description                   |
| -------- | -------- | ----------------------------- |
| SQLite   | Markdown | Export DB to Markdown files   |
| Markdown | SQLite   | Import Markdown files into DB |

### Migration stats

After migration, the CLI prints:

```
Migration: N from sqlite, M from markdown, K imported
```

- `from_sqlite` / `from_markdown`: Entries read from source
- `imported`: Successfully written to target
- `skipped`: Entries with missing key or content
- `errors`: Store failures

### SQLite requirement

When SQLite is disabled (`SC_ENABLE_SQLITE` off), only Markdown ↔ Markdown migrations are supported.
