# Harden Infrastructure Command

Chain these steps:

1. Use `infrastructure-hardening` to develop hardening guidelines for servers and systems
2. Use `network-segmentation` to implement network isolation
3. Use `kubernetes-security` to harden Kubernetes clusters if applicable
4. Use `certificate-management` to audit and secure certificate management

Deliverables:

- OS hardening guide (baseline configuration)
- Patching roadmap and schedule
- Network segmentation implementation plan
- Kubernetes hardening checklist
- Certificate inventory and renewal plan

After completion, suggest follow-up commands: `audit-cloud`, `review-iam`.
